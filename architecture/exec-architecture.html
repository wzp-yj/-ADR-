当前代码路径、Shadow 能力与未来接口分开标记。认知可以判断和提出候选，但任何外部执行仍需要两次确认。

Real：当前已运行 Shadow：运行并记录，不影响回复 Future：只有接口或设计 Safety Gate：必须由用户确认

Layer 3 / Cognitive Layer

Cognitive Engine 认知引擎

判断是在聊天、讨论、灵感、决定、任务、执行意图还是情绪表达。Assessment 不参与回复生成；已确认的 Inspiration Policy 只投影 pending 候选。
Turn → ContextResolver → CognitiveEngine (Shadow) → CognitiveAssessment → pending InspirationCandidate → user review → Inspiration → event / scheduled Evolution scan → pending EvolutionProposal → user review → Relation / versioned IdeaDirection
shadow real Hybrid Engine

real CognitiveAssessment
不可变 revision、项目快照、证据摘要、脱敏审计与只读历史 API。
real Inspiration Projector
重新验证持久 Assessment、项目快照和 artifact；幂等生成 pending 候选，失败不影响回复。
future Semantic Provider
可接云模型、本地 vLLM 或其他 Provider，不能改写项目解析结果。
灵感权威池与可审核的创造性分支
real Inspiration Pool

real Idea Evolution Engine
PostgreSQL Retriever + deterministic / optional semantic Hybrid；manual、event、scheduled 扫描只生成 pending Proposal。
confirm real Evolution Review
Expo 展示证据时间线、不确定性、影响和当前版本；接受后才建立 Relation 或追加不可变 IdeaDirection 版本。
accepted Inspiration → immutable scan marker → one-shot worker + advisory lock → bounded corpus → grounded proposal → reject / snooze / accept → unchanged sources + auditable derived state
边界保证：Assessment 不修改 Assistant 回复；Projector 和 Evolution scan 只能创建 pending 候选；拒绝或稍后处理不产生派生状态；IdeaDirection 转任务仍必须进入独立 ExecutionTaskDraft 确认流。

Layer 4 / Safety Gate 1

第一次确认 ExecutionTaskDraft 范围确认

Cognitive task artifact 判定后生成 Draft。用户可编辑项目、标题、目标和验收标准，确认后才创建 ExecutionTask。Agent 在确认前不会被分配或调用。
Cognitive task artifact ? idempotent Draft projection ? scope edit (versioned) ? user accept/reject ? confirmed ExecutionTask snapshot
real TaskDraftProjector
从 CognitiveAssessment task artifact 判定生成 pending Draft，同一 artifact 幂等，失败不影响 Assistant 回复。
real Draft Edit & Decision
用户可编辑项目、标题、目标、验收标准。accept 创建 Task，reject 不创建 Task。accept 幂等。
confirm real Scope Review
Expo 展示来源摘要、建议能力和不可编辑的系统守卫。缺项目/目标/验收标准时禁止接受。
接受前不会分配或调用 Agent。拒绝不创建 Task。projector 失败不影响 Assistant 回复。Draft / Task 不含 Provider 原始推理。

Layer 5 / Isolated Agent Runtime

Agent Registry → Worktree / Container → Change + Test Report

插件发现和 Agent 契约已实现。Shell + Test + Git + LLM Coder Agent 运行于 Git Worktree 隔离环境。插件接口可替换。Agent 只能修改隔离副本，不能直接写用户项目。
real Shell Agent
在 Git Worktree 隔离环境执行 Shell 命令，通过 git diff 捕获变更。
real Test Agent
在隔离环境运行 pytest，报告测试结果。
real Git Agent
status, diff, commit, branch, tag — 所有操作限定在隔离 worktree。
real LLM Coder Agent
调用 OpenAI 兼容 LLM 生成代码，写入隔离 workspace，解析 FILE/END 块输出 Change。
real Codex Agent (双模式)
CLI/API 双模式。本地 CLI 不可用时自动回退 OpenAI 兼容 API。Claude Code Agent 待接入。
