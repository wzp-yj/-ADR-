8 个子系统 — 建议执行顺序

P0

基础架构 & 项目管理

FastAPI 骨架、PostgreSQL 模型、项目 CRUD、Agent 插件接口。所有后续子系统的基础。

P1

持续对话 + 长期记忆

会话管理、流式响应、pgvector 记忆存储与检索。最核心的用户交互闭环。

P2

上下文管理 + 灵感池

歧义澄清引擎、项目前置信度判断、灵感快速捕获与状态流转。

P3

多 Agent 协作 + 审核中心

Agent 调度器、插件化接入、Diff 展示、用户确认-执行-回滚流程。

P4

多模态输入

语音转文字、PDF/Word/Excel 解析管道。进度上与前三阶段可部分并行。
